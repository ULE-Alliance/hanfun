1 # HAN-FUN
2 
3 This project aims at providing an implementation of the [ULE Alliance](http://www.ulealliance.org/) HAN-FUN application specification.
4 
5 ## Overview
6 
7 A HAN-FUN network is composed by a concentrator (base) and one or more nodes, in a star configuration,
8 all the nodes connect to the concentrator that serves a router between all the nodes in the network.
9 
10 A HAN-FUN device, either concentrator or node, is composed of one or more units. Each unit implements
11 a profile, i.e. a predefined configuration of interfaces and any number of optional interfaces.
12 
13 HAN-FUN interfaces have two roles, _Client_ and _Server_ and define what attributes are present in each
14 role. They also define the commands supported between each role.
15 
16 In each device there is a special unit, unit 0, that provides core infrastructure services, for
17 example, the __Device Management__ service, the __Device Information__ service and the
18 __Attribute Reporting__ service.
19 
20 In order to access the HAN-FUN network, a node must first register it self with the base to obtain
21 it's HAN-FUN address, by using the commands available in the Device Management service.
22 
23 This address will then be used to send/receive packets from the network.

25 ## Project Setup
26 
27 This project is implemented in C++ and depends only on the standard C++ library for usage, however
28 the unit tests and example applications depend on third-party libraries.
29 
30 The build system used is [CMake](http://www.cmake.org/).
31 
32 * Dependencies
33 
34  * gcc (>= 4.6)
35  * cmake (>= 2.8.11)
36 
37 * Building:
38 
39  $ mkdir build/
40  $ cd build/
41  $ cmake ..
42  $ make
43  $ make doc # Build the documentation

72 ## Examples
73 
74 The folder _apps/_ contains two example applications, one for the concentrator (base) side and one
75 for the node side.
76 
77 The _apps/example_XX.cpp_ files contain standalone applications that demo API usage.
78 Use __make example_XX__ to build each application or __make examples__ to build all of them.
79 
80 The applications communicate over TCP/IP.
81 The support for this transport layer is based on the [libuv](https://github.com/joyent/libuv).
82 This is downloaded and build automatically when building the example applications is requested.
83 
84 To build the example applications, issue the following commands (on the root of the project):
85 
86  $ mkdir build
87  $ cd build
88  $ cmake -DHF_BUILD_APPS=ON ..
89  $ make
90 
91 This will build the example applications for the base (_build/apps/base_) and for
92 the node (_build/apps/node_).
93 
94 The *base* application demonstrates the registration of nodes in a HAN-FUN network and the binding
95 of nodes that have complementary interfaces.
96 
97 The *node* application demonstrates, a simple switch or a simple light application.
98 
99 For example to register a _simple switch_ and _simple light_ to the _base_ and bind the _switch_ to
100 the _light_ so that when the _switch toggles_ it toggles the _light_.
101 
102  * Open three terminals and start the applications;
103 
104  $ apps/base (Base)
105  $ apps/node 1 (Simple Switch)
106  $ apps/node 2 (Simple Light)
107 
108  * Change the first node application to a simple switch;
109 
110  > ss
111 
112  * Change the second node application to a simple light;
113 
114  > sl
115 
116  * Start registration on the base and register the simple switch;
117 
118  > r 1 1 (on the base terminal)
119  > r (on the switch terminal)
120  > a (check device address, it should be 1)
121 
122  * Start registration on the base and register the simple light;
123 
124  > r 1 2 (on the base terminal)
125  > r (on the light terminal)
126  > a (check device address, it should be 2)
127 
128  * Bind the _switch_ to the _light_;
129 
130  > b 1 2 (on the base terminal)
131 
132  * Send a toggle command from the _switch_ to the _light_ :
133 
134  > toggle (on the switch terminal)
135 
136 The last command should make the node application configured as a _light_ print that a
137 toggle command was received.
138 
139 Please see the help in the applications for further features.
